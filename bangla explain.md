# ক্যাপস্টোন প্রজেক্ট প্রস্তাবনা — ৫টি আইডিয়া (বিস্তারিত বাংলা ব্যাখ্যা)

**লেখক প্রসঙ্গ:** উত্তর দক্ষিণ বিশ্ববিদ্যালয় (NSU) — ইলেকট্রিক্যাল অ্যান্ড কম্পিউটার ইঞ্জিনিয়ারিং (ECE) ফাইনাল ইয়ার ক্যাপস্টোন প্রজেক্টের জন্য প্রস্তাবনা প্যাক।

## প্রজেক্টগুলো কেন দরকার — বড় ছবি

### বাংলাদেশ ও GCC-তে আমরা কোন সমস্যায় জীবন যাপন করি?

| সমস্যা | দৈনন্দিন জীবনে প্রভাব | কোন প্রজেক্ট সাহায্য করবে |
|--------|----------------------|---------------------------|
| **ধীর ইন্টারনেট (2G/3G)** | ভিডিও কল কেটে যায়, পিক্সেল দেখা যায় | প্রস্তাবনা ১, ২, ৪, ৫ |
| **IMO/WhatsApp যথেষ্ট নয়** | কল চলে কিন্তু মুখ স্পষ্ট নয়; কেন বাজে বোঝা যায় না | প্রস্তাবনা ২, ৫ |
| **RMG কারখানায় তাপ** | শ্রমিক অসুস্থ, উৎপাদন বন্ধ, আইনি ঝুঁকি | প্রস্তাবনা ৩ |
| **গ্রামের স্কুলে শিক্ষক কম** | গণিত/বিজ্ঞান পড়ানোর মান নিম্ন | প্রস্তাবনা ৪ |
| **AI চ্যাটবট ব্যবহারের ঝুঁকি** | ChatGPT-তে ব্যক্তিগত কথা দিলে ডেটা বিদেশে চলে যায় | প্রস্তাবনা ৫ (সব প্রজেক্টে নীতি) |

### ক্যাপস্টোন প্রজেক্ট মানে শুধু অ্যাপ বানানো নয়

একটি **বিশ্বমানের বিশ্ববিদ্যালয়ের** ক্যাপস্টোনে শিক্ষক ও নিয়োগকর্তা যা দেখতে চায়:

1. **স্পষ্ট সমস্যা** — কার জীবনে কী বদলাবে?
2. **পরিমাপযোগ্য ফল** — VMAF, stall rate, citation accuracy, alert F1 ইত্যাদি
3. **বেসলাইন তুলনা** — H.265, WhatsApp, নিয়ম-ভিত্তিক সিস্টেমের সাথে
4. **ব্যর্থতার বিশ্লেষণ** — কোথায় কাজ না করবে, সৎভাবে লিখা
5. **ওপেন সোর্স + ডেমো** — GitHub, ভিডিও, লাইভ লিংক
6. **নৈতিকতা ও গোপনীয়তা** — বিশেষ করে শিশু ও শ্রমিক ডেটা

আমাদের পাঁচ প্রস্তাবনা এই মানদণ্ড মেটাতে ডিজাইন করা হয়েছে।

## RAG চ্যাট কী এবং কেন

### সাধারণ ChatGPT vs RAG — সহজ বাংলায়

**সাধারণ ChatGPT (বিনা RAG):**
- মডেল তার **প্রশিক্ষণের মemory** থেকে উত্তর দেয়
- আপনার কারখানার সেন্সর ডেটা, আজকের ক্লাসের স্লাইড, আপনার আগের কলের ইতিহাস **জানে না**
- কখনো **গল্প বানিয়ে** (hallucination) উত্তর দিতে পারে
- আপনার প্রশ্ন **তৃতীয় পক্ষের ক্লাউডে** চলে যায়

**RAG (Retrieval-Augmented Generation):**
1. ব্যবহারকারী প্রশ্ন করেন
2. সিস্টেম **আপনার অনুমোদিত উৎস** থেকে খুঁজে নিয়ে আসে:
   - আপনার **পূর্বের চ্যাট**
   - **লাইভ টেলিমেট্রি** (বিটরেট, তাপমাত্রা, সেন্সর)
   - **অফিসিয়াল ডকুমেন্ট** (NCTB, ILO, FAQ)
3. **স্ব-হোস্ট LLM** (Ollama) শুধু খুঁজে পাওয়া টেক্সট থেকে উত্তর বানায়
4. প্রতি উত্তরে **উৎস দেখায়**: `[chat:2026-06-08]` বা `[ilo_heat_03]`

### RAG চ্যাটের ৬ নিয়ম (সব প্রজেক্টে)

| নিয়ম | ব্যাখ্যা |
|-------|---------|
| **স্কোপ** | শুধু এই ব্যবহারকারী বা এই গ্রুপের ডেটা |
| **সম্মতি** | প্রথম ব্যবহারে জানানো: “সহকারী আপনার এনক্রিপ্টেড চ্যাট ইতিহাস ব্যবহার করবে” |
| **পাবলিক API না** | OpenAI/Claude বন্ধ — Ollama/স্ব-হোস্ট |
| **এনক্রিপশন** | চ্যাট বিশ্রামে AES-256; ট্রানজিটে TLS |
| **মুছে ফেলার অধিকার** | “আমার মেমোরি মুছুন” — ইনডেক্স + মেসেজ মুছে যাবে |
| **উৎস দেখানো** | উত্তরে কোন চ্যাট/ডক থেকে — কপি-পেস্ট পুরো চ্যাট নয় |

## পূর্বের চ্যাট ডেটা থেকে RAG

### কেন পূর্বের চ্যাট মেমোরি দরকার?

**উদাহরণ — ভিডিও কল (প্রস্তাবনা ২):**
- জুন ৮: ব্যবহারকারী জিজ্ঞেস করেছিল “ভিডিও কেন ব্লকি?”
- সহকারী বলেছিল: “110 kbps, 12% loss — অডিও-অনলি মোড চালু করুন”
- জুন ১৫: আবার একই সমস্যা
- **পূর্বের চ্যাট RAG ছাড়া:** সহকারী প্রতিবার নতুন করে শুরু করবে
- **পূর্বের চ্যাট RAG সহ:** “জুন ৮-এর মতোই — অডিও-অনলি চালু করছি [chat:2026-06-08]”

**উদাহরণ — সিকিউর মেসেঞ্জার (প্রস্তাবনা ৫):**
- “আহমেদ কী বলেছিল মিটিং টাইমে?”
- RAG শুধু আহমেদের সাথে আপনার এনক্রিপ্টেড চ্যাট থেকে `[msg_58]` খুঁজে বের করে
- উত্তর: “আহমেদ বলেছিল বৃহস্পতিবার ৩টা [msg_58, ১০ মার্চ]”

### পূর্বের চ্যাট RAG কীভাবে কাজ করে (ধাপে ধাপে)

```
ধাপ ১: প্রতিটি চ্যাট মেসেজ এনক্রিপ্ট করে সেভ
ধাপ ২: প্রাইভেট সার্ভারে (বা ফোনে) embedding তৈরি — তৃতীয় পক্ষে না যায়
ধাপ ৩: নতুন প্রশ্ন আসলে সিমিলার পুরোনো মেসেজ খুঁজে (vector search)
ধাপ ৪: শুধু top-3 রিলেটেড চ্যাট চাঙ্ক + লাইভ ডেটা LLM-কে দেয়
ধাপ ৫: LLM উত্তর + উৎস সাইটেশন দেয়
```

### গুরুত্বপূর্ণ: Embedding লিক হতে পারে

এমনকি এনক্রিপ্টেড চ্যাটের embedding যদি পাবলিক API-তে যায়, **অর্থের অর্থ** আংশিকভাবে লিক হতে পারে। তাই:
- Embedding **স্ব-হোস্ট** বা **ডিভাইসে**
- পাবলিক এজেন্টে **পুরো চ্যাট এক্সপোর্ট** কখনো না

## প্রস্তাবনা ১ — নিউরাল ভিডিও কম্প্রেশন

### পূর্ণ নাম

**রিট্রিভাল-গ্রাউন্ডেড সেমান্টিক ভিডিও কমিউনিকেশন আল্ট্রা-লো বিটরেট মোবাইল নেটওয়ার্কের জন্য**

### সমস্যা — বিস্তারিত

**বাংলাদেশে ভিডিও কলের বাস্তব চিত্র:**
- গ্রামীন 2G/3G: 50–200 kbps স্থির নয় — ঝটপট উঠানামা
- ঢাকার মোবাইল ডেটা: রাশ hour-এ নেটওয়ার্ক জ্যাম
- GCC প্রবাসী: সস্তা Wi-Fi কিন্তু ভিড়ে লিংক দুর্বল

**H.265 / H.264 কী করে:**
- পিক্সেল কম্প্রেস করে — 200 kbps-এ মুখ **ব্লক ব্লক** দেখায়
- নেটওয়ার্ক loss হলে পিক্সেল আরও বিকৃত

**সমস্যার মূল:** কম বিটরেটে **মুখ স্পষ্ট, ঠোঁটের চলাচল** জরুরি — বিশেষ করে ভিডিও কলে। পিক্সেল-পারফেক্ট পুরো স্ক্রিন জরুরি নয়।

### সমাধান — কী বানাবেন

**তিনটি অংশ:**

#### অংশ A: সেমান্টিক এনকোডার
- ভিডিও থেকে **অর্থবহ তথ্য** নিক্ষেপ: মুখের ভঙ্গি, ঠোঁট, চোখ
- বিটরেট: **150–200 kbps** লক্ষ্য
- পুরো স্ক্রিনের প্রতিটি পিক্সেল না — **মানে-ভিত্তিক** সংকুচিত রিপ্রেজেন্টেশন

#### অংশ B: রিট্রিভাল-অগমেন্টেড ডিকোডিং
- একটি **ভিজুয়াল মেমোরি ব্যাংক** (vector database)
- মিলে যাওয়া **প্যাচ/সিন** খুঁজে নিয়ে ভিডিও **পুনর্গঠন** করবে

## প্রস্তাবনা ২ — লো-ব্যান্ডউইথ ভিডিও কল

### পূর্ণ নাম

### সমস্যা — বিস্তারিত

**ব্যবহারকারীর অভিজ্ঞতা:**
- IMO/WhatsApp কল চলে কিন্তু **কেন বাজে** — কেউ বোঝায় না
- “ভিডিও ফ্রিজ” → ব্যবহারকারী অ্যাপ বা নেটওয়ার্ককে দোষ দেয় — **তথ্য নেই**
- একই ব্যক্তি প্রতিদিন ভিন্ন নেটওয়ার্ক — **আগের সমাধান মনে রাখা** দরকার

**টেকনিক্যাল সমস্যা:**
- WebRTC ডিফল্ট সেটিং সব নেটওয়ার্কের জন্য এক নয়
- বিটরেট, packet loss, jitter, RTT — ব্যবহারকারী দেখতে পায় না

### সমাধান — কী বানাবেন

#### অংশ A: WebRTC ভিডিও কল
- 1:1 কল
- **অ্যাডাপ্টিভ বিটরেট** — লক্ষ্য 100–200 kbps
- Bangladesh 3G / GCC trace-এ টেস্ট

#### অংশ B: টেলিমেট্রি বাস
- রিয়েল-টাইম: kbps, loss %, RTT, resolution, stall count
- প্রতিটি কলের `call_id` — ব্যক্তিগত ইতিহাস

#### অংশ C: RAG কল সহকারী চ্যাট (মূল ফিচার)

| ব্যবহারকারী জিজ্ঞেস করেন | RAG কোথা থেকে খুঁজে |
|-------------------------|---------------------|
| “ভিডিও কেন ঝাপসা?” | লাইভ টেলিমেট্রি + FAQ + **পূর্বের চ্যাট/কল** |
| “গতকাল অডিও-অনলি বলেছিল — এখন করো” | **পূর্বের চ্যাট** + বর্তমান kbps |
| “মঙ্গলবারের মতো একই সমস্যা” | **পূর্বের চ্যাট রিট্রিভাল** |

**উদাহরণ উত্তর:**

#### অংশ D: গোপনীয়তা
- চ্যাট **E2E এনক্রিপ্ট** (বা TLS + এনক্রিপ্টেড স্টোর)
- **পাবলিক এজেন্ট না** — Ollama স্ব-হোস্ট VPS
- শুধু **আপনার** কল ইতিহাস — অন্য কার না
- “চ্যাট মেমোরি মুছুন” বাটন

### IMO প্রতিদ্বন্দ্বী হিসেবে কোথায় খেলবেন

**ব্যবস্থাপনিক প্রভাব:**

- **অন-প্রেম** — কারখানা LAN-এ Ollama — চ্যাট ক্লাউডে না

## প্রস্তাবনা ৫ — টেলিগ্রাম-টাইপ সিকিউর অ্যাপ

### পূর্ণ নাম

**সিকিউর লো-ব্যান্ডউইথ মেসেঞ্জার প্রাইভেট অন-ডিভাইস রিট্রিভাল-অগমেন্টেড মেমোরি সহ (টেলিগ্রাম-গ্রেড প্রাইভেসি)**

### সমস্যা — বিস্তারিত

**কেন নতুন অ্যাপ দরকার:**

১. **টেলিগ্রাম/WhatsApp চ্যাট এনক্রিপ্ট** — কিন্তু **AI সহকারী নিরাপদ নয়**
   - Telegram বট → API ক্লাউডে
   - WhatsApp Meta AI → পাবলিক
   - আপনার **পুরো চ্যাট ইতিহাস** প্রশ্নে যায়

২. **পূর্বের চ্যাট মেমোরি = সবচেয়ে উপকারী + সবচেয়ে ঝুঁকিপূর্ণ**
   - “আহমেদ কী বলেছিল?”
   - “গত মাসে নেটওয়ার্ক সমস্যা কী কী?”
   - পাবলিক API = **সম্পূর্ণ জীবন বিদেশে**

৩. **ধীর নেটওয়ার্কে এক অ্যাপ চাই**
   - সিকিউর চ্যাট + (ঐচ্ছিক) লো-বিটরেট কল
   - **প্রাইভেট AI** যে **মনে রাখে** — OpenAI-তে না পাঠিয়ে

### সমাধান — কী বানাবেন

#### অংশ A: E2E এনক্রিপ্ট মেসেঞ্জার
- রেজিস্ট্রেশন, কন্টাক্ট, **1:1 চ্যাট**
- Signal Protocol বা libsodium — **সার্ভার প্লেইনটেক্সট পড়তে পারবে না**
- Forward secrecy — সেশন কী পরিবর্তন

#### অংশ B: প্রাইভেট RAG — পূর্বের চ্যাট মেমোরি (মূল ফিচার)

```
ব্যবহারকারী: “আহমেদ মিটিং টাইমে কী বলেছিল?”

১. শুধু আহমেদের সাথে এনক্রিপ্টেড ইনডেক্স খোঁজা
২. top-3 মেসেজ: [msg_42], [msg_58], [msg_71]
৩. প্রাইভেট LLM উত্তর:
   “আহমেদ বলেছিল বৃহস্পতিবার ৩টা [msg_58, ১০ মার্চ]”
৪. কখনো: পুরো চ্যাট এক্সটার্নাল API-তে না
```

**অন্যান্য RAG প্রশ্ন:**
- “এই মাসে নেট সমস্যার সারাংশ”
- “আমি কখন বলেছিলাম অডিও-অনলি চালু করো?”
- “আমার আপলোড করা PDF নোট থেকে খুঁজো”

#### অংশ C: পাবলিক এজেন্ট **আর্কিটেক্টurally ব্লক**
- অ্যাপ কনফিগে OpenAI/Claude endpoint **ব্ল্যাকলিস্ট**
- শুধু Ollama / স্ব-হোস্ট vLLM
- ডেমো: “পাবলিক ChatGPT vs আমাদের প্রাইভেট মেমোরি”

#### অংশ D: ঐচ্ছিক লো-বিটরেট কল
- প্রস্তাবনা ২-র সাবসেট — 200 kbps
- কল সহকারী + চ্যাট মেমোরি **এক ইকোসিস্টেম**

## শিক্ষকের জন্য আবেদন পত্র

সম্মানিত সুপারভাইজার,

আমি উত্তর দক্ষিণ বিশ্ববিদ্যালয় ইলেকট্রিক্যাল অ্যান্ড কম্পিউটার ইঞ্জিনিয়ারিং বিভাগের ক্যাপস্টোন প্রজেক্টের জন্য **পাঁচটি প্রস্তাবনা** পেশ করছি। সকল প্রস্তাবনার সাধারণ থিম: **উন্নয়নশীল অঞ্চলের বাস্তব সমস্যা** (ধীর নেট, RMG তাপ, গ্রামীন শিক্ষা, ডিজিটাল গোপনীয়তা) এবং **RAG-চালিত চ্যাট সহকারী**, যা উত্তর দেয় **এনক্রিপ্টেড পূর্বের চ্যাট**, লাইভ টেলিমেট্রি (যেখানে প্রযোজ্য), এবং **অনুমোদিত দলিল** থেকে — সাধারণ LLM মেমোরি থেকে নয়।

**গুরুত্বপূর্ণ নীতি:** সংবেদনশীল চ্যাট এবং পূর্বের-চ্যাট RAG-এর জন্য **পাবলিক এজেন্ট API (OpenAI, Claude ইত্যাদি) ব্যবহার করা হবে না**, কারণ এতে ব্যক্তিগত বার্তা, কল মেটাডেটা, শ্রমিক বা শিশু সম্পর্কিত তথ্য তৃতীয় পক্ষের ক্লাউডে চলে যায়। আমরা **স্ব-হোস্ট Ollama/vLLM** এবং **E2E এনক্রিপ্টেড স্টোরেজ** ব্যবহার করব।

| প্রস্তাবনা | সংক্ষিপ্ত বিবরণ |
|-----------|----------------|
| **১** | ML: সেমান্টিক ভিডিও + রিট্রিভাল ডিকোডিং (&lt;200 kbps) + প্রাইভেট সাপোর্ট চ্যাট |
| **২** | সিস্টেম: লো-ব্যান্ডউইথ ভিডিও কল + RAG কল সহকারী (পূর্বের চ্যাট মেমোরি) |
| **৩** | IoT: RMG কারখানা তাপ-স্ট্রেস মনিটরিং + RAG সেফটি সহকারী |
| **৪** | EdTech: গ্রামীন 2G টেলি-টিউটরিং + NCTB-গ্রাউন্ডেড বাংলা RAG |
| **৫** | সিকিউরিটি: টেলিগ্রাম-গ্রেড E2E মেসেঞ্জার + প্রাইভেট past-chat RAG (পাবলিক API ব্লক) |

আপনার নির্দেশনা চাই: ক্যাপস্টোন স্কোপ, ল্যাব রিসোর্স, এবং মূল্যায়ন মানদণ্ডের সাথে কোন প্রস্তাবনা সবচেয়ে উপযুক্ত। অনুমোদনের পর **একটি প্রাথমিক প্রস্তাবনা** নির্দিষ্ট করে কাজ শুরু করব।
